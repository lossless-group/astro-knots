# astro-knots/splash

GitHub Pages splash for the [astro-knots](https://github.com/lossless-group/astro-knots) pseudomonorepo.

Live: <https://lossless-group.github.io/astro-knots/>

## What this is

A small Astro site that:

1. Greets visitors with a hero, the candid "what worked / what didn't" reading of the workspace, and a curated gallery of the 12 sibling sites under `sites/`.
2. Demonstrates the **two-tier token + three-mode contract** that astro-knots is built on, with a live mode toggle in the header — toggle it and the entire splash pivots between light, dark, and vibrant.
3. Renders the parent's `context-v/` and (when synced) each sibling site's `changelog/` and `context-v/` as readable archives.
4. Deploys for free off GitHub Pages on push to `main`.

It is **not** the eventual marketing site. The directory is named `splash/` precisely to leave that linguistic space open.

Implements the habit at [`lossless-monorepo/context-v/habits/Maintain-a-Github-Splash-Page-for-each-Repo.md`](../../context-v/habits/Maintain-a-Github-Splash-Page-for-each-Repo.md).

## Local dev

```bash
pnpm install --ignore-workspace
pnpm dev
```

The dev server respects `base: '/astro-knots/'`, so visit <http://localhost:4321/astro-knots/>.

> `--ignore-workspace` is required because the parent `astro-knots/pnpm-workspace.yaml` doesn't include this site; the splash installs its own deps independently.

## The signature: live mode toggle

The header on every page exposes a three-mode toggle (light · dark · vibrant). It writes `data-mode` on `<html>` and persists the choice in `localStorage` under `astro-knots-mode`. The `theme.css` file is structured per the **Astro Knots two-tier token convention**:

- **Tier 1 — named tokens** (BEM-ish: `--color__indigo-electric`, `--font__display`) — raw values, mode-invariant.
- **Tier 2 — semantic tokens** (kebab-case: `--color-bg`, `--color-accent`) — referencing the named tier; rebound per `data-mode='light' | 'dark' | 'vibrant'`.

The TokenSwatchRow component on the index renders semantic tokens directly so a visitor can watch the lattice shift in real time. This isn't decoration — it's the convention every sibling site implements, demonstrated in its own splash.

## The roll-up flow

The splash aggregates each sibling site's `changelog/` and `context-v/` into the workspace feed — but the build itself doesn't fetch anything. **A human runs `pnpm rollup:sync` deliberately**; the rolled-up content lands in `splash/src/rollup/`, gets committed, and from then on the build is pure file IO.

```bash
pnpm rollup:sync
git add src/rollup/
git commit -m "sync(rollup): refresh from <reason>"
git push
```

This is a deliberate design choice (vs. fetching at every build):

| | Sync-on-build | Sync-on-demand (current) |
|---|---|---|
| Build time | 30s+ (API calls) | ~1s (file IO) |
| CI auth | needs `GITHUB_TOKEN` | none |
| Rate limits | every build counts | sync runs at human cadence |
| Diff visibility | invisible | git diff shows what rolled up |
| Determinism | depends on remote state | what you commit is what deploys |

### When to run `rollup:sync`

- A sibling site shipped a noteworthy `changelog/` entry you want surfaced.
- You bumped a submodule pointer.
- Periodically (e.g. weekly) to catch upstream drift.

### Auth for the sync

The sync script reads `GITHUB_TOKEN` or `GITHUB_API_TOKEN` from your shell or `splash/.env`:

```bash
# splash/.env
GITHUB_API_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxx
```

Or grab it from your `gh` CLI:

```bash
echo "GITHUB_API_TOKEN=$(gh auth token)" >> splash/.env
```

Anonymous mode works at 60 req/hr; one full sync uses ~60–80 calls, so you'll have to wait an hour between syncs without a token.

## Where things live

| Path | Purpose |
|---|---|
| `src/pages/index.astro` | Hero, sites grid, two-tier token demo, "what worked / what didn't" candor, latest changelog teaser |
| `src/pages/changelog/` | List + detail for the union of `../changelog/` (local) and `src/rollup/changelog/` (synced sites) |
| `src/pages/context-v/` | List + detail for `../context-v/` ∪ `src/rollup/context-v/` |
| `src/content/site-highlights/*.md` | Curated site cards. Edit these to update the gallery. |
| `src/rollup/` | Synced submodule content. Generated by `pnpm rollup:sync`. **Do not edit by hand** — gets wiped on next sync. |
| `src/content.config.ts` | Collection definitions; uses a small `unionLoader` to read local + rollup. |
| `src/styles/theme.css` | The two-tier token system + three-mode contract — the centerpiece convention this splash demonstrates. |
| `src/loaders/` | The fetcher (`rollupFetch.ts`), GitHub Content API client, `.gitmodules` parser, and minimal frontmatter parser. |
| `src/components/KnotMark.astro` | Animated interlocking-thread mark used in the hero, topbars, and footer. |
| `src/components/ModeToggle.astro` | The live three-mode switcher. |
| `src/components/SiteCard.astro` | One site in the gallery, with a brand-color thread strip. |
| `src/components/TokenSwatchRow.astro` | Live demo of the two-tier system — pivots with the mode toggle. |
| `scripts/rollup-sync.ts` | The CLI script invoked by `pnpm rollup:sync`. |

### Path aliases

`tsconfig.json` declares short aliases so loader code never needs `../../../` guesswork: `@components/*`, `@layouts/*`, `@loaders/*`, `@content/*`, `@pages/*`, `@styles/*`, `@lib/*`, `@/*`. Use them.

## Curating the site gallery

Each site gets one markdown file under `src/content/site-highlights/`. Frontmatter:

```yaml
---
title: hypernova-site               # display name
lede: One-sentence pitch.            # appears under the title
order: 40                            # integer, lower sorts first; alphabetical tiebreak by filename
status: Active                       # Active | In progress | Stable | Beta | Alpha | Experiment
repo: https://github.com/...         # repo URL
live_url: https://hypernova.io       # optional; if present, card links here
icon: 🌌                             # emoji or path under public/
featured: true                       # featured cards take a wider tile in the grid
thread_color: "#5b6fff"             # optional override; defaults to --thread__<slug> from theme.css
maintained_by: Hypernova Labs        # optional attribution line
tags: [Client, Brand-kit, Three-modes]
---

Long-form description in markdown. Renders as the card's expanded body once the gallery
adds detail rendering (currently only frontmatter is shown).
```

`order` ties are broken alphabetically by filename — never throws. Per-site brand colors are declared in `theme.css` under `--thread__<slug>` so the `SiteCard` can pick them up automatically; override per-card via `thread_color: "#..."` in frontmatter.

## Deploy

`.github/workflows/pages.yml` (in the parent repo root) builds this site on push to `main` and deploys via `actions/deploy-pages@v4`. Configure GitHub Pages on the `astro-knots` repo to **"GitHub Actions"** as the source.

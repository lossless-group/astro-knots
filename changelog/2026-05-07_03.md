# Changelog - 2026-05-07 (#3)

## v1 ship: YouTube playlist embed family functional

## Why Care?

The Lossless Flavored Markdown video-component family is the first part of LFM that crossed from "auto-unfurl ships you a working iframe" to "auto-unfurl ships you a thoughtful, theme-aware, design-system-aligned UI surface." The playlist case was the proving ground because it's the hardest one — a single-video embed has one rendering decision; a playlist has a dozen (item list, sort, click-to-play, lazy-load, mobile fallback, vertical alignment, scroll affordances, ...). All of those are now wired and working in v1.

This entry exists mostly to capture **the full day's session arc** as a single referenceable table — the work split across two changelog entries (`_01` for the first-pass playlist landing, `_02` for the cross-cutting `aside-escape` pattern) plus this third one for the ship close-out. Six commits in the site repo, four in the parent.

## What this v1 ships

A bare `youtube.com/playlist?list=...` URL on its own line in any markdown file — *or* a `:::youtube-playlist` container directive with the URL pasted inside, *or* `::youtube-playlist{id="..."}` as a leaf directive — auto-unfurls into:

- A **player card** that flows in the article reading column at the column's natural width, with a header strip carrying playlist title, channel, item count, and an "Open on YouTube" CTA.
- A **scroller aside** that escape-positions itself in the article's right margin track at ≥1280px viewports (or stacks below at 768–1279px, or surfaces as a full-screen panel below 768px).
- **Build-time data acquisition** via the YouTube Data API v3 — playlist metadata, paginated item list, batched per-item durations — cached at `src/data/youtube-playlist-cache.json` (gitignored) with 30-day TTL. ~3 quota units per playlist on cold cache; free tier covers hundreds of playlists per build.
- **Sort by `addedAt` descending** so the most-recently-added video sits on top, regardless of how the playlist owner has the list ordered on YouTube.
- **Click-to-play** via a small Svelte island that hydrates with `client:visible`, finds the iframe by ID, and uses `playVideoAt(N)` through the YouTube IFrame Player API.
- **Wheel and touchpad scroll** that actually works inside the scroller (today's bug fix), themed with a thin brand-aqua scrollbar.
- **Sticky "Up 5 / Down 5" buttons** at the top and bottom of the scroll viewport, auto-disabled at scroll edges.
- **Active-item highlight** (brand-aqua left border) that updates as the iframe advances.
- **Vertical centering** of the scroller against the player, so when the scroller is taller than the iframe it extends equally above and below.

## Two cross-cutting patterns established, available to the next sidebar component

1. **`MobileFeaturePlaceholder.astro`** — the wrapper that lets any sidebar-positioned LFM component degrade gracefully on phones to a placeholder + full-screen panel. Spec §5.9. First consumer is the playlist scroller; future consumers (`LinkPreview__Article--Card` with `aside="right"`, `LinkRollup__Column`) inherit the contract.

2. **The `aside-escape` CSS recipe** — `position: absolute` + `left: calc(100% + gap)` + `width: min(maxw, calc(((100vw - 100%) / 2) - 2rem))` against a `position: relative` wrapper, lets a child element pop out of a centered article column into the right margin track. Spec §4.1.3.5. The orchestrator pattern (one wrapper composing in-column + escape-positioned siblings) is the reusable shape; `MarginEscape.astro` extraction waits for a second consumer per the "premature extraction is worse than waiting one round" principle.
